## 背书节点(endorser peer) - 执行交易 - org1: a=100 - org2: a=200 - 以上两笔交易在背书节点的执行结果都是正确的
# 排序节点 - 排序节点看到的顺序: a=100;a=200
# 记账节点 - 乐观锁 - 读写集 - {read: a=0, write: a=100}; {read: a=0, write:a=200} - a会被赋值为100
```shell $ peer chaincode instantiate ... ``` 以上命令将会调用智能合约里的`Init`函数 ```shell $ peer chaincode invoke ... ``` 以上命令将会调用`Invode`函数

### 对等节点分类 - 背书节点: 安装了chaincode的peer节点 - 记账节点: 所有节点都是记账节点 - 领导节点: 排序节点广播区块时先广播给领导节点，再由每个组织的领导节点广播给自己组织的对等节点 - 锚节点: 和锚节点同一个组织的节点可以通过锚节点来发现属于同一个通道的其他组织的节点
